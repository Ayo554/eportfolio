Secure Software Development

In this module, we’ll take an in depth look at the security risks which are associated with programming languages, from the perspectives of design and architecture approaches, programming paradigms, testing, the impact and implications of operating systems and libraries on software development, the security risks associated with distributed systems and APIs, and future trends in secure software and systems design. Running through these topics will be the concept of the secure software development life cycle.

Module Aims and Learning Outcomes

In this module, we shall:

  • Acquire a critical understanding of the concept of abstraction in programming.
  • Develop an understanding of the basic principles of secure development methodologies.
  • Explore how to undertake analysis, program design, software construction and testing required for software development.
  • Demonstrate an understanding of the basic principles of architecture, as well as traditional and contemporary Software Development Life Cycle (SDLC) models, such as TOGAF and Agile.
  • The ability to put into practice the techniques learned in a team environment, demonstrating how to deal with conflicts and how to make compromises, and be able to critically evaluate the effectiveness of the approaches.
  • Utilise the opportunity to reflect on and evaluate your own personal development.

On completion of this module, you will be able to::

  • Identify and manage security risks as part of a software development project.
  • Critically analyse development problems and determine appropriate methodologies, tools and techniques (including program design and development) to solve them.
  • Design and develop/adapt computer programs and to produce a solution that meets the design brief and critically evaluate solutions that are produced.
  • Systematically develop and implement the skills required to be effective member of a development team in a virtual professional environment, adopting real-life perspectives on team roles and organisation.

Unit 3 - Introduction to Programming Languages

Development Team Project: Design Document

Project Brief:

There have recently been a number of accusations of copying and copyright infringement in the music industry. Several industry stakeholders and performers have
suggested that a digital application is required to manage copyright material and to ensure that writers and performers rights are protected.
You have been commissioned to design an application that can be used as a secure enclave to store lyrics, music scores and musical recordings
(e.g. digital audio in formats such as MP3, FLAC and so on) on behalf of copyright owners and performers. It should provide a database to store the artefacts,
be able to generate a checksum to confirm the integrity of the artefact, the ability to encrypt the artefacts while they are stored, plus a mechanism
to generate and store time stamps that can be used to record creation and modification dates and times. The application should support different roles
such as administrators that can create and delete artefacts, and users that can add, view and modify their own entries and view but cannot modify those of others.

Your design should be captured using a UML tool and should consist of at least three different types of UML diagrams.
Your submission will be graded on the quality of the design, plus how well it adheres to UML standards and conventions.

It should detail the design of the database, data structures and data flows as well as application details expressed as a class diagram.
Your design should demonstrate an effective working knowledge of object-oriented design principles such as encapsulation, inheritance, and abstraction, amongst others.

Unit 6 - Future trends in Secure Software Development

Development Individual Project - Coding Output

Project brief:

You are now required to implement the application which you designed in Unit 3.
This should be written in Python and should not use or be based on commonly available frameworks such as Django, Flask and so on.
It should be a command line interface application. In addition, you should explain and justify the use of external libraries and
code that account for more than 20% of your application, marks will be deducted for over-reliance on external libraries or code.
For example – do not use frameworks such as Django where your original code would usually comprise much less than 30% of the overall code.

You should add comments to your code, which illustrate, amongst other qualities, how your application implements the design patterns you created in the Unit 3 Assignment.
The implementation should illustrate CRUD functionality – i.e., how to create, read, update, and delete artefacts. It should also provide and demonstrate multiple roles
such as administrator and user.

The main purpose of this assessment is to implement a secure application, so emphasis should be placed on secure coding practices and
procedures, including the use of security testing applications; marks will be awarded for evidence of the use of such tools.
